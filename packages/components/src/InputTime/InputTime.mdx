# InputTime

<ComponentStatus stage="pre" responsive="no" accessible="no" />

Select a time of day. For example the time a messages should be sent or when a
job started/ended.

At the moment the 12-hour vs 24-hour preference is set by the operating system
and this component doesn't look at the Jobber setting.

## Simple Example

```ts
import { InputTime } "@jobber/components/InputTime";
```

## Events

If the input starts empty the `ChangeEvent` isn't fired until a full time is
set. Setting just the hour or minute or _meridiam_ won't fire the event. When
the last element is set the change event will fire.

<Playground>
  {() => {
    const [time, setTime] = useState(new CivilTime(3, 52));
    const resetTime = () => {
      setTime(new CivilTime(3, 52));
    };
    const handleChange = newTime => {
      setTime(newTime);
    };
    return (
      <>
        <InputTime value={time} onChange={handleChange} />
        <pre>{time.toString()}</pre>
        <Button label="Reset" onClick={resetTime} />
      </>
    );
  }}
</Playground>
