# Heading

<ComponentStatus stage="pre" responsive="no" accessible="yes" />

Headings are used as the titles of each major section of a page in the
interface. Reserved for short, important text or numbers.

## Page title

This should explain the main subject of the page. There should only be 1 on a
page.

<Playground>
  <Heading variation="page">New client</Heading>
</Playground>

## Category

Used to categorize large groups of content. For example, there are 2 main
category when creating a new client. Client details and property details.

This heading can be skipped if there's no large groups that need breaking up.

<Playground>
  <Heading variation="category">Client details</Heading>
  ...
  <Heading variation="category">Property details</Heading>
  ...
</Playground>

## Topic

Used to group content and forms on a single topic

<Playground>
  <Heading variation="topic">Contact details</Heading>
  ...
  <Heading variation="topic">Notifications</Heading>
  ...
  <Heading variation="topic">Additional details</Heading>
  ...
</Playground>
